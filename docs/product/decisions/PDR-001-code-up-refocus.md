# PDR-001: Adopt Code-Up Intent Refocus

## Status

Accepted for beta candidate.

## Context

The precursor product could model local requirements, but the suite needs a
member that answers why code exists. The refocus changes Plainweave's center
from broad requirements management to a code-grounded intent corpus.

## Decision

Plainweave beta centers on the graph `SEI -> requirement -> goal`. Requirements
remain local and shell-friendly, but the product value is the readable corpus
and the orphan/trace queries around code.

## Reversal Trigger

Reverse or narrow this decision if a beta dogfood run cannot produce useful
answers for a representative public code surface without duplicating Loomweave
or Legis authority.
