# Plainweave Product Vision

Plainweave holds code-grounded intent. Its job is to make the reason for a code
entity visible, queryable, and attributable without taking authority away from
the tools that own code identity or policy decisions.

## Serves

- Agents making or reviewing code changes.
- Maintainers curating the requirement corpus.
- Suite tools that need local intent facts without a release verdict.

## Refuses

- It does not replace Loomweave SEI identity.
- It does not replace Legis enforcement or audit.
- It does not decide that a requirement is satisfied merely because work closed.
- It does not treat agent-authored bindings as accepted human truth.

## Authority Grant

Plainweave may create local goals, requirements, bindings, and read models inside
this repo. Outward-facing actions such as suite admission, publication, final
name, hub roster changes, and sibling obligations require owner approval.
