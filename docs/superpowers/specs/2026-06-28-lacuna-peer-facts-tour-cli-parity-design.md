## Mission

Plainweave 1.1 shipped two local-first, advisory peer-facts producers as **MCP-only**
surfaces. This work (a) closes the CLI/MCP parity gap by exposing both producers'
full `.v1` envelopes over `plainweave … --json`, and (b) extends the Lacuna
cross-member tour to **demonstrate and regression-protect** them via two new matrix
cells: `plainweave+wardline` and `plainweave+warpline`.

This is a **harness-completeness / regression bet**, not a correctness fix (the
producers are already contract- and unit-tested in Plainweave) and **not a release
blocker**. Part A is nonetheless a real, user-facing Plainweave product change.

## Verified ground truth (from source, 2026-06-28)

**Plainweave CLI** — `argparse`; flat commands registered in
`src/plainweave/cli_commands.py::register_commands()` (dossier pattern:
`subparsers.add_parser(name, …)` → `.add_argument(…)` → `.set_defaults(handler=fn)`).
`--json` is always `action="store_true"`. Console entry `plainweave.cli:main`.

**Reuse target** — `PlainweaveMcpSurface(root)` (`src/plainweave/mcp_surface.py:391`,
ctor `__init__(self, root: Path | None = None)`) already returns the **exact full
`.v1` envelope** for both producers:
- `plainweave_wardline_peer_facts_list(*, limit=50, offset=0)` (`mcp_surface.py:751`)
  → delegates to `WardlineAdapter.list_peer_facts` (`wardline_adapter.py:242`).
  Reads `.wardline/*-findings.jsonl` only; **does not require an initialized
  `.plainweave/` store** (`_project_key()` → `inspect_project` returns
  `project_key: None` on absence, never raises; `mcp_surface.py:825`,
  `cli_commands.py:1095`).
- `plainweave_requirements_enrichment_get(*, entity_refs)` (`mcp_surface.py:759`)
  → logic lives in the surface itself (no service method). Goes through
  `_result` (`mcp_surface.py:724`), which **catches `PlainweaveError` and returns a
  clean error envelope** (e.g. `NOT_FOUND` when the store is uninitialized).

Both surface methods therefore always return a well-formed envelope dict (success or
error); neither raises to the caller in normal operation.

**Cycle hazard** — `mcp_surface` imports from `cli_commands`, so CLI handlers MUST
import `PlainweaveMcpSurface` **locally inside the handler body** (the repo's
established pattern, mirroring the local `web` import in `cli.py:19`). A module-level
import would create a circular import.

**No-verdict validator** — test-only modules (`tests/wardline_contract.py:84`
`validate_wardline_peer_facts` + `:68` `assert_no_wardline_verdicts`;
`tests/warpline_contract.py:47` `validate_requirements_enrichment` + `:35`
`assert_no_warpline_verdicts`). `validate_*` takes the **bare `data` payload** (it
asserts a closed key set, so the envelope wrapper must be stripped);
`assert_no_*_verdicts` recurses over anything. "Run the validator over CLI output too"
= call them in the new CLI tests — exactly the precedent in
`tests/test_cli_intent_coverage.py`.

**Contract note** — `authority_boundary` lives **inside `data`** (validator-enforced),
not envelope-level as the older spec §6.4 drawing shows. We build no envelope by hand;
we pass the surface's envelope through verbatim, so this is automatically correct.

**Lacuna tour** — a "cell" is just a string in a lacuna's `demonstrates` list in
`tour/lacunae.toml`; `Manifest.cells()` → `docs/matrix.md` (auto-generated by
`make tour`; never hand-edited). A "capability fact" is a `(rule_token, qualname)`
tuple in `StepResult.surfaced`. `make verify` reds unless every **live** lacuna is
surfaced by a step emitting a pair whose `rule == lacuna.expected_rule` and whose
qualname suffix-matches `lacuna.symbol` (`tour/report.py:44-52`), AND `docs/tour.md` +
`docs/matrix.md` byte-match the regenerated output. Steps are registered explicitly in
`_drive()` (`tour/__main__.py:26-45`). The single mockable CLI seam is
`_plainweave_json(args)` (`tour/steps.py:648`). Determinism: `detail` is frozen prose,
`surfaced` carries only stable `(token, qualname)` pairs, live numbers ride the
non-rendered `note`.

**Lacuna `.wardline/` is unusable as-is** — a volatile, actively-rotated 20-snapshot
window with **no scan-identity manifest**; it cannot drive a deterministic, manifest-
based resolved/unseen demo. We generate a frozen fixture instead (below).

## Part A — Plainweave CLI parity

Two new flat subcommands in `src/plainweave/cli_commands.py`, each reusing the MCP
surface and printing its envelope.

### `plainweave wardline-peer-facts [--json] [--limit N] [--offset N]`
- Registration: `wardline-peer-facts` parser, `--limit type=int default=50`,
  `--offset type=int default=0`, `--json action="store_true"`,
  `set_defaults(handler=handle_wardline_peer_facts)`.
- Handler: local-import `PlainweaveMcpSurface`; build `PlainweaveMcpSurface(project_root())`;
  call `plainweave_wardline_peer_facts_list(limit=args.limit, offset=args.offset)`;
  emit the envelope and return the right exit code (see "envelope emission" below).
- Works with no `.plainweave/` store (the producer reads `.wardline/` only).
  Out-of-range pagination → the surface returns a `VALIDATION` error envelope → exit 2.

### `plainweave requirements-enrichment <entity_ref>... [--json]`
- Registration: `requirements-enrichment` parser, positional `entity_ref nargs="+"`,
  `--json action="store_true"`, `set_defaults(handler=handle_requirements_enrichment)`.
- Handler: local-import `PlainweaveMcpSurface`; call
  `plainweave_requirements_enrichment_get(entity_refs=args.entity_ref)`; emit + exit.
- Accepts either a `loomweave:eid:…` SEI or a legacy dotted locator (the surface
  canonicalizes locally). Uninitialized store → clean `NOT_FOUND` error envelope → exit 2.

### Envelope emission (shared helper)
A small `handle_*` body, or a shared `_emit_surface_envelope(args, envelope)` helper:
- `--json` → `print(json.dumps(envelope))` (full envelope, matching `_handle_output`).
- without `--json` → `print(json.dumps(envelope["data"]))` on success; on an error
  envelope, mirror `_emit_error`'s human form (`f"{code}: {message}"`).
- Exit code: `0` if `envelope["ok"]`, else `4` if `error.code == INTERNAL` else `2`
  (mirrors `_emit_error`). A defensive `try/except PlainweaveError → _emit_error`
  wraps the surface call as belt-and-suspenders (the surface shouldn't raise, but the
  handler stays correct if a future change makes it).

### Part A tests (TDD, in-process `main([...])` + `capsys`, mirroring
`test_cli_intent_coverage.py`)
- `tests/test_cli_wardline_peer_facts.py`:
  - happy path over a planted single-snapshot `.wardline/` in `tmp_path` (no store):
    assert `schema == "weft.plainweave.wardline_peer_facts.v1"`, `ok is True`;
    `validate_wardline_peer_facts(envelope["data"])` + `assert_no_wardline_verdicts(envelope)`.
  - absent `.wardline/` → `data["freshness"] == "unavailable"`, degraded
    `wardline_findings_absent`, exit 0 (unavailable is a valid advisory result).
  - `--limit 0` (out of range) → error envelope `VALIDATION`, exit 2.
  - non-`--json` prints `data` (still valid JSON).
- `tests/test_cli_requirements_enrichment.py`:
  - seed a bound + orphan + unresolvable mix (reuse the `_seed_bound` pattern from
    `tests/test_warpline_requirements_enrichment.py`); assert `schema`, `ok`, the
    three statuses, `validate_requirements_enrichment(data)` + `assert_no_warpline_verdicts(envelope)`.
  - uninitialized store → `NOT_FOUND` error envelope, exit 2.

## Part B — Lacuna tour demos

Two new step functions in `tour/steps.py`, two new `[[lacuna]]` entries in
`tour/lacunae.toml`, registered in `_drive()` after `plainweave_intent()`. Both follow
the tour contract (never raise; degrade to `ok=False`). New frozen-anchor constants in
the `PLAINWEAVE_*` block (`steps.py:640`).

### Demo 1 — `pw-requirements-enrichment` (cell `plainweave+warpline`)
Runs at ROOT after self-seeding (`plainweave_seed.seed(pw)` — idempotent, wipes+rebuilds
`.plainweave/`), then `plainweave requirements-enrichment <ref…> --json`. Three frozen
locator anchors:
- COVERED `python:function:specimen.cli._add_book` → `status == "present"`, non-empty `requirements`.
- ORPHAN `python:function:tour.__main__.main` → `status == "absent"`.
- UNRESOLVABLE `python:function:specimen.cli._does_not_exist` → `status == "unavailable"`
  (**load-bearing: NOT `absent`** — the no-silent-clean invariant; an identity gap is
  "cannot tell", never "definitively none").

Emits `("pw-requirements-enrichment", "specimen.cli._add_book")` **only if all three
statuses hold**, so a regression that collapses `unavailable → absent` (or vice-versa)
drops the pair → `make verify` reds. Lacuna entry: `expected_rule =
"pw-requirements-enrichment"`, `symbol = "specimen.cli._add_book"`, `file =
"specimen/cli.py"`, `demonstrates = ["plainweave", "plainweave+warpline"]`,
`expected_tool = "plainweave"`, `expected_kind = "finding"`.

### Demo 2 — `pw-wardline-peer-facts` (cell `plainweave+wardline`) — FULL resolved/unseen
A new `tour/wardline_peerfacts_seed.py` writes a **frozen two-snapshot fixture** into a
gitignored staged dir (e.g. `tour/.peerfacts/`), mirroring how `plainweave_seed.py`
rebuilds `.plainweave/`. Content is frozen Python constants → byte-identical every run,
so no committed `.wardline/` fighting Lacuna's `.gitignore`, and full determinism.

Fixture (`.wardline/` inside the staged dir; the adapter reads only `kind`,
`ruleset_id`, `scope.covered_paths` from each `scan_manifest` line):

- `20260101T000000Z-findings.jsonl` (prior) — manifest `ruleset_id="demo-ruleset-1"`,
  `scope.covered_paths=["specimen/peerfacts.py","specimen/legacy_area.py"]`; findings:
  - `fp-active` defect, `specimen/peerfacts.py`, qualname anchor (kept in latest).
  - `fp-nondefect` kind `fact`, `specimen/peerfacts.py` (kept in latest).
  - `fp-resolved` defect, `specimen/peerfacts.py` (GONE from latest → resolved/unseen).
  - `fp-outofscope` defect, `specimen/legacy_area.py` (GONE; path ∉ latest covered →
    indeterminate + honest `wardline_scope_mismatch`).
- `20260102T000000Z-findings.jsonl` (latest) — manifest `ruleset_id="demo-ruleset-1"`
  (same → no ruleset-mismatch), `scope.covered_paths=["specimen/peerfacts.py"]`;
  findings: `fp-active`, `fp-nondefect` only.

Yields: `freshness="current"`; `facts=[active defect, non-defect]`;
`resolved_or_unseen=[fp-resolved]`; `summary.indeterminate>=1`; `degraded` contains
`wardline_scope_mismatch`. The leg runs `plainweave wardline-peer-facts --json` with
`cwd=<staged dir>`, then a second invocation with `cwd=<a dir without .wardline/>`
asserting `freshness=="unavailable"` + degraded `wardline_findings_absent`.

Asserts (all must hold to emit the pair):
1. an `active`, defect (`non_defect is False`) fact surfaces (qualname anchor);
2. a `non_defect is True` fact surfaces;
3. `resolved_or_unseen` non-empty (item at `specimen/peerfacts.py`);
4. `wardline_scope_mismatch` present in `degraded` (scope honestly flagged);
5. absent `.wardline/` → `freshness == "unavailable"` (**never clean**).

Emits `("pw-wardline-peer-facts", "<qualname anchor>")`. Lacuna entry: `expected_rule =
"pw-wardline-peer-facts"`, `symbol = <qualname anchor>`, `demonstrates = ["plainweave",
"plainweave+wardline"]`, `expected_tool = "plainweave"`, `expected_kind = "finding"`.
The qualname anchor and `file` will reference a real specimen symbol/path if
`tour/docs_gen.py` requires `lacuna.file` to exist (verified in implementation);
otherwise a synthetic `specimen/peerfacts.py` anchor is used.

### Part B mechanics
- Extend the `_plainweave_json` seam with an optional `cwd` parameter (default ROOT) so
  the wardline leg can target the staged fixture dir while staying the single mockable
  seam (tests monkeypatch it).
- Unit tests `tests/test_steps_plainweave_peerfacts.py` (mirroring
  `tests/test_steps_plainweave.py`): monkeypatch `_plainweave_json` to return canned
  envelopes; assert each leg emits its pair only when its full predicate holds, and
  degrades (`ok=False`, no pair) on any missing/`unavailable`/wrong-status branch.
- Narrative: extend the frozen `detail=` prose of the two new legs; regenerate
  `docs/tour.md` + `docs/matrix.md` via `make tour` (the new cells appear from
  `demonstrates`); commit both. Gitignore the staged fixture dir.

## Invariants preserved end-to-end
- **Advisory only, zero verdict vocabulary** — we pass the surface envelopes through
  unchanged; CLI tests run `assert_no_*_verdicts` over the output.
- **Local-first** — no live peer calls; `authority_boundary.live_peer_calls is False`,
  `local_only is True` (already in the envelopes).
- **No-silent-clean** — the demos assert it, not just happy paths: unresolvable →
  `unavailable` not `absent`; absent `.wardline/` → `unavailable` not clean;
  out-of-scope prior finding → `indeterminate` + `wardline_scope_mismatch`, never a
  silent "resolved".
- **Deterministic, frozen anchors** — no hardcoded hex SEIs; refs are stable dotted
  locators; the wardline fixture is frozen constants.

## Decisions / defaults (non-forks, recorded for the approval gate)
1. **Reuse the MCP surface directly** from CLI handlers (not a new service method) —
   the `requirements_enrichment` logic only exists in the surface; this is minimal and
   matches the tasking.
2. **Validator-based CLI tests** (+ schema/exit-code asserts), not vendored CLI
   goldens — the MCP wire-goldens already byte-pin the envelopes; CLI parity only needs
   to prove it emits the same envelope verdict-free.
3. **Tour-time generated wardline fixture** (frozen constants → staged gitignored dir),
   not a committed `.wardline/` — sidesteps Lacuna's `.wardline` gitignore and mirrors
   the existing `plainweave_seed.py` rebuild pattern. Equally deterministic.
4. **Wardline demo = FULL resolved/unseen** (owner-selected) — two snapshots + manifests,
   resolved/unseen + honest scope-mismatch, plus absent→unavailable.

## Folded-in adjacent changes (provenance: sibling product contract)
Two small, already-tested Plainweave changes are present in the working tree, landed as
part of another product's contract, and ship with this work by owner decision (green:
`tests/test_doctor.py` + `tests/test_warpline_requirements_enrichment.py`, 19 passed):
1. `mcp_surface.py::plainweave_requirements_enrichment_get` drops `state == "rejected"`

   Part B's enrichment demo** (a rejected-only entity reads `absent`, not a silent
   `present`). +2 tests.
2. `cli_commands.py::_doctor_wardline_check` makes the remediation `next_action`
   root-aware (`cd <root> && wardline scan .` when `root != cwd`). +2 tests.

These are noted in the PDR/CHANGELOG as included-and-attributed; they are not re-derived
